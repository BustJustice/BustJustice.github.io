离开上海回到二线城市

发表于 2020-11-22 分类于 心得体会 评论数：
为什么来为什么走

  • 大学四年自己学习计算机专业，自学过java，php，sql，javascript等编程语言，
    互联网行业风生水起，心中萌生了要去北上广做一名大厂程序员，发挥自己所学的想法
    但生活是生活，毕业以后，我顺利进入微盟成为一名软件工程师
    我在脑中模拟着未来的生活，发现了不对劲的地方

  • 首先，上海的高房价，高入户门槛决定了我一定留不住，
    上海半年的开支包括房租、水电、吃穿等消费让自己的高工资攒不下钱，
    哪怕未来承担更多的压力换来了更高薪，然而二线的房子总是会涨，自己如果不早回二线
    恐怕工作几年后也没有回头路可以走，人生只有一次，也因为年轻选错了还可以重来，
    我决定离开上海回到二线发展

未来去往哪里

  • 二线城市并没有太多程序员开发的岗位，我已经明白大概率不能继续做开发了，但无需后悔
    我依然感谢微盟曾经让我加入开发工作，也感谢领导对我的谆谆教诲，在自己身上的学习能力，
    项目的组织协调能力，模块管理的能力不会丢，这些是基础，也是各行各业都需要的

  • 希望进入国企，公务员，事业编制等单位，然后工作定居。我明白企业需要员工长久稳定的工作，
    员工也希望自己稳定下来，是高房价阻碍了这个目标的达成。因此我也希望能在二线城市完成这个目标，
    自己已然成长，渐渐成了家里的顶梁柱，父母接近退休，自己的选择需要为这个家考虑，我不希望他们
    继续担忧下去，我重新规划了人生，这次我希望往这条路走至少十年。

上海一行的收获
  • 首先是技术上的严谨，这源于领导同事的对我的代码严格检查，
    让我明白代码不仅仅要实现这个功能还要足够精简，
    没有一丝无用的成分，更不要轻易重构项目，高度耦合代码，因为要考虑到维护成本
  • 了解规则，尊重规则。作为程序员要有程序员的职业操守，程序员不止是写代码，
    也要在项目的规则体系下，才能发挥出最大的功效
  • 心怀感恩，但行好事。自己收到了微盟支付的薪水，受到了来自领导的经验教诲，
    我发自内心感谢为我所作的这一切。
    在我离开之后，在外人问起时，我会回答到微盟是家好公司，我遇到的是位好领导。
    自己心中问心无愧，坦坦荡荡

容易被遗忘的几个函数

原帖见此:https://blog.csdn.net/qq_39132756/article/details/85007082
reduce,reduceRight

语法：array.copyWithin(target , start , end)
方法用于从数组的指定位置拷贝元素到数组的另一个指定位置中，会覆盖原有成员
target ：必需。从该位置开始替换数据。
start ：可选。从该位置开始读取数据，默认为 0 。如果为负值，表示倒数。
end： 可选。到该位置前停止读取数据，默认等于数组长度。如果为负值，表示倒数。

use strict有哪些影响

发表于 2019-08-24 评论数：

详见：阮一峰日志
http://www.ruanyifeng.com/blog/2013/01/javascript_strict_mode.html

常见影响

1.使用变量必须显示声明var let
2.禁止with语句
3.创建eval作用域
4.禁止this指向全局对象
5.禁止删除变量
6.不得重名属性方法
7.禁止八进制表示
8.对arguments对象限制
9.严格模式新增了一些保留字：
implements, interface, let, package, private, protected, public, static, yield。
使用这些词作为变量名将会报错。

网络攻击的几种方式

发表于 2019-08-24 评论数：

DNS欺骗(域名污染)

简单的说就是改变了域名解析指向的IP地址
详情见：https://blog.csdn.net/qq754406613/article/details/43649007

XSS攻击

简单的说就是通过插入恶意脚本，实现对用户游览器的控制。
详情见：https://www.cnblogs.com/bangerlee/archive/2013/04/06/3002142.html

DDos攻击

详情见：https://www.cnblogs.com/2myroad/articles/3881042.html
SYN Flood攻击利用了TCP三次握手的缺陷，能够以较小代价使目标服务器无法响应
DNS Query Flood就是攻击者操纵大量傀儡机器，对目标发起海量的域名查询请求。
HTTP Flood（CC攻击）攻击者通过匿名代理对攻击目标发起HTTP请求。

域名劫持

详情见：https://www.jb51.net/network/353770.html
域名服务器上都会保存一大堆的域名记录（每条记录包含“域名”和“IP地址”）。
当收到域名查询的时候，域名服务器会从这堆记录中找到对方想要的，然后回应给对方。
如果域名服务器上的某条记录被【人为修改】了（改成错的），
那么一旦要查询这条记录，得到的就是错误的结果。这种情况称之为“域名劫持”。
